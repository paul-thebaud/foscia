# Types

## Utilities types

:::tip
Many utilities types are used by FuncClient but are not exported to avoid
polluting your code or conflicting with your own utilities types.
You may copy those types as you want and use them in your project.
:::

### `Constructor<T>`

```typescript
type Constructor<T> = new (...args: any[]) => T;
```

### `Dictionary<T>`

```typescript
type Dictionary<T = unknown> = { [K: string]: T };
```

### `Arrayable<T>`

```typescript
type Arrayable<T> = T | T[];
```

### `Awaitable<T>`

```typescript
type Awaitable<T> = T | Promise<T>;
```

## Model types

### `ModelId`

```typescript
type ModelId = string | number;
```

### `ModelConfig`

```typescript
type ModelConfig = {
  type: string;
  baseURL?: string;
  comparator?: (newValue: unknown, prevValue: unknown) => boolean;
  cloner?: <T = unknown>(value: T) => T;
};
```

### `ModelDefinition`

A model definition (an object with attributes/relations/custom properties).

*This type does not really exist, it is only here for documentation purpose*.

### `ModelSchema<D>`

An extracted schema (only attributes and relations) from a model definition `D`.

### `ModelClass<D>`

A model class made with the model factory and defined by its definition `D`.

### `ModelInstance<D>`

A [`ModelClass<D>`](#modelclassd) instance.

### `Model<D, I>`

A [`ModelClass<D>`](#modelclassd) and constructor of a
[`ModelInstance<D> I`](#modelinstanced).

### `ModelValues<M>`

An inferred schema's values dictionary for a [`ModelClass<D>`](#modelclassd)
or a [`ModelInstance<D>`](#modelinstanced).

### `ModelKey<M>`

An inferred schema's value's key for a [`ModelClass<D>`](#modelclassd)
or a [`ModelInstance<D>`](#modelinstanced).

### `ModelRelationDotKey<M>`

An inferred schema's relation's key for a [`ModelClass<D>`](#modelclassd)
or a [`ModelInstance<D>`](#modelinstanced). Key might be a direct relation
or a deeper relation (up to 5 depth) with a dot separation.

Examples for a `Post` model: `comments`, `comments.author`,
`comments.reactions`, etc.

## Action types

### `ActionMethod`

An HTTP method as a fully lower or upper cased string (GET, POST, etc.).

### `ActionContext`

A variable and extendable context to be enhanced and ran by an action.

```typescript
export type ActionContext = {
  method?: ActionMethod;
  baseURL?: string;
  type?: string;
  id?: ModelId;
  relation?: string;
  includes?: string[];
  path?: string;
  params?: Dictionary<any> | string;
  payload?: unknown;
  [key: string]: unknown;
};
```
