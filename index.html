REST and JSON:API ready

FuncClient comes with a set of tools to quickly integrate with a REST or JSON:API backend.
You may also create your own adapters to integrate with any data sources (Soap, SQL, IndexedDB, etc.).

Modular and fully tree-shakable

With functional programming, FuncClient can be used in many ways to fit your needs and unused functions can be tree-shaken from your production build automatically.

Strongly typed

FuncClient was build at start with TypeScript to propose you a secure and clean typings experience with consistent generic types.

Define your models

FuncClient provides a simple way to define your models, which may combine attributes, relations or custom properties and methods.

import { makeModel, attr, hasOne } from 'func-client/core';
import type User from './User';
import type Tag from './Tag';

export default class Post extends makeModel('posts', {
  title: attr<string>(),
  content: attr<string>(),
  author: hasOne<User>(),
  tags: hasMany<Tag>(),
  get titleWithAuthor() {
    return `${this.title} by ${this.author.name}`;
  },
}) {}

Play with your models

Once your models are ready, you can define an action factory which you will use to run actions over your models. FuncClient proposes many functions to run action, from classic CRUD to custom actions...

import { find, update, include, oneOrFail, oneOrCurrent } from 'func-client/core';
import Post from './post';
import action from './action';

const post = await action()
  .use(find(Post, 123))
  .use(include('author', 'tags'))
  .run(oneOrFail());

fill(post, { title: 'Hello World!' });

const updatedPost = await action()
  .use(update(post))
  .run(oneOrCurrent());
