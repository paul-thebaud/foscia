Types

Model types​

ModelId​

type ModelId = string | number;

ModelSchema<D>​

An extracted schema (only attributes and relations, not extension) from a model definition D.

ModelClass<D>​

A model class made with the model factory and defined by its definition D.

ModelInstance<D>​

A ModelClass<D> instance.

Model<D, I>​

A ModelClass<D> and constructor of a ModelInstance<D> I.

ModelValues<M>​

An inferred schema's values dictionary for a ModelClass<D> or a ModelInstance<D>.

ModelKey<M>​

An inferred schema's value's key for a ModelClass<D> or a ModelInstance<D>.

ModelRelationDotKey<M>​

An inferred schema's relation's key for a ModelClass<D> or a ModelInstance<D>. Key might be a direct relation or a deeper relation (up to 5 depth) with a dot separation.

Examples for a Post model: comments, comments.author, comments.reactions, etc.

Action types​

ActionMethod​

An HTTP method as a fully lower or upper cased string (GET, POST, etc.).

ActionContext​

A variable and extendable context to be enhanced and ran by an action.

export type ActionContext = {
  method?: ActionMethod;
  baseURL?: string;
  type?: string;
  id?: ModelId;
  relation?: string;
  includes?: string[];
  path?: string;
  params?: Dictionary<any> | string;
  payload?: unknown;
  [key: string]: unknown;
};

Utilities types​

tip

Many utilities types are used by FuncModel but are not exported to avoid polluting your code or conflicting with your own utilities types. You may copy those types as you want and use them in your project.

Arrayable<T>​

type Arrayable<T> = T | T[];
